InfraGmen-Goku (インフラGメン)
ネットワークの「ZEN（秩序）」を守る、リアルタイム通信監視・判定支援ツール
🚀 概要
本ツールは、OSが提供する公開情報（psutil）を用いて、現在確立されているネットワーク通信をリアルタイムに一覧表示し、人間による判断を支援するための可視化ツールです。

ブラックボックス化しがちなバックグラウンド通信を「見て理解できる形」に整理し、インフラエンジニアが「未知の接続」や「非標準ポートの利用」をいち早く察知してセルフデバッグすることを目的としています。

✨ 特徴
リアルタイム・プロセス特定: 通信を行っているプロセス名（実行ファイル名）を瞬時に特定して表示。
インテリジェント・フィルタリング:
ホワイトリスト: 信頼できる通信を「通常タブ（緑）」へ。
ブラックリスト: 警戒すべき通信を「警告タブ（赤背景）」で強調表示。
未登録WEB: 443番（HTTPS）は「青ハイライト」で分離し、未知のサイトをあぶり出し。
ZEN整列アルゴリズム: 日本語（全角文字）が含まれても、各カラムが垂直に美しく揃う独自パディング処理を実装。
実戦的な安定動作設計: DNSキャッシュ機能を搭載し、システム負荷を抑えつつ高速なホスト名解決を実現。再起動不要でCSV設定をリアルタイム反映します。

🛡️ 設計思想（ポリシー）
本ツールは 「自動遮断・自動判定」を行いません。
通信を止めない: 既存の通信環境に影響を与えません。
内容を覗かない: パケットの中身（ペイロード）の傍受・解析は一切行いません。
判断を奪わない: あくまで「人間が状況を把握し、判断するための材料を揃える」ことに特化しています。
IDS/IPSの代替ではなく、**運用・保守・開発現場における「判断支援」**にフォーカスした設計です。

🛠 技術スタック
Language: Python 3.11+
Libraries:
psutil (プロセス・ネットワーク情報取得)
tkinter (GUI)
threading (非同期監視処理)
unicodedata (表示幅の動的計算)

⚠️ 免責事項・注意事項
本ツールは学習および個人のネットワーク管理を目的として作成されています。
管理者権限がない環境では、一部のプロセス名が取得できない場合があります。
業務利用時は、必ず所属組織のセキュリティポリシーに従ってください。

✍️ 作成背景
業務や開発の現場において「通信が多すぎて、結局何が起きているか分からない」「自動ツールの判断根拠が見えない」という課題を感じたことから作成しました。 可視化と判断の間にある“溝”を埋め、エンジニアが自身のテリトリーを完全に把握できる「秩序」を提供することを目指しています。
